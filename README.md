## Architecture Overview

```
SubSentinel/
├── main.py                    # CLI entry point
├── core/
│   ├── scanner.py             # Main orchestration engine
│   ├── config.py              # Centralized configuration
│   └── models.py              # Data models (Finding, DNSData, etc.)
├── modules/
│   ├── enumerator.py          # Multi-source subdomain enumeration
│   ├── dns_analyzer.py        # Async DNS resolution & dangling detection
│   ├── http_analyzer.py       # HTTP/HTTPS response analysis
│   ├── fingerprinter.py       # Service fingerprinting engine
│   ├── claimability.py        # Claimability verification (false positive reduction)
│   ├── screenshot.py          # Playwright screenshot capture
│   ├── evidence.py            # Structured evidence collection
│   └── confidence.py          # Confidence scoring engine
├── fingerprints/
│   └── fingerprints.yaml      # 25+ provider fingerprint database
├── alerts/
│   └── manager.py             # Telegram/Discord/Slack/Webhook/Email alerting
├── monitoring/
│   └── watcher.py             # Continuous monitoring daemon
├── reports/
│   └── generator.py           # JSON/HTML/Markdown/CSV report generation
├── ai/
│   └── detector.py            # Future ML detection module (architecture ready)
└── utils/
    ├── database.py            # SQLite persistence
    ├── logger.py              # Rich logging
    ├── proxy.py               # Proxy rotation
    └── rate_limiter.py        # Token-bucket rate limiting
```

## Installation

### Prerequisites
- Python 3.9+
- pip

### Quick Install

```bash
# Clone / download SubSentinel
https://github.com/sapariya333-sys/SubSentinel
cd SubSentinel

# Create virtual environment (recommended)
python3 -m venv venv
source venv/bin/activate   # Linux/Mac
# venv\Scripts\activate    # Windows

# Install dependencies
pip install -r requirements.txt

# Install Playwright browsers (for screenshots)
playwright install chromium

# Optional: Install external tools for enhanced enumeration
# subfinder:  https://github.com/projectdiscovery/subfinder
# amass:      https://github.com/owasp-amass/amass
```

## Usage

### Basic Scan

```bash
# Single domain (interactive legal prompt)
python main.py -d example.com

# Accept legal disclaimer non-interactively (CI/CD use)
python main.py -d example.com --accept-legal

# Scan a list of domains
python main.py -l domains.txt

# Use pre-enumerated subdomains (skip enumeration)
python main.py -s subdomains.txt
```

### Output Formats

```bash
# JSON report (always generated by default)
python main.py -d example.com --json

# HTML interactive report
python main.py -d example.com --html

# All formats
python main.py -d example.com --json --html --markdown --csv

# Custom output directory
python main.py -d example.com -o /tmp/results
```

### Enumeration Sources

```bash
# Use all sources (requires tools installed)
python main.py -d example.com --subfinder --amass

# With Chaos API key
python main.py -d example.com --chaos --chaos-key YOUR_KEY
```

### Performance Tuning

```bash
# High-speed scan (increase concurrency)
python main.py -d example.com --threads 100 --rate-limit 200

# Conservative scan (avoid detection/rate limits)
python main.py -d example.com --threads 10 --delay 0.5

# Custom timeout and retries
python main.py -d example.com --timeout 15 --retries 5
```

### OPSEC / Stealth Options

```bash
# Single proxy
python main.py -d example.com --proxy http://127.0.0.1:8080

# Proxy rotation list
python main.py -d example.com --proxy-list proxies.txt

# Custom user agent
python main.py -d example.com --user-agent "Mozilla/5.0 (custom)"
```

### Alerting

```bash
# Telegram alerts
python main.py -d example.com \
  --telegram-token YOUR_BOT_TOKEN \
  --telegram-chat YOUR_CHAT_ID

# Discord webhook
python main.py -d example.com \
  --discord-webhook https://discord.com/api/webhooks/...

# Slack
python main.py -d example.com \
  --slack-webhook https://hooks.slack.com/services/...

# Email
python main.py -d example.com \
  --email security@example.com \
  --smtp-host smtp.example.com

# Generic webhook (JSON POST)
python main.py -d example.com \
  --webhook https://your-siem.example.com/ingest

# Minimum severity for alerts
python main.py -d example.com --min-severity critical
```

### Continuous Monitoring

```bash
# Monitor a domain every hour (3600s)
python main.py -d example.com --watch

# Custom interval (every 30 minutes)
python main.py -d example.com --watch --interval 1800

# Full monitoring with Telegram alerts
python main.py -d example.com \
  --watch \
  --interval 3600 \
  --telegram-token TOKEN \
  --telegram-chat CHATID \
  --accept-legal
```

### Confidence Filtering

```bash
# Only report high-confidence findings (default: 30)
python main.py -d example.com --min-confidence 60

# Only critical findings
python main.py -d example.com --min-confidence 85
```

### Fingerprint Management

```bash
# Update fingerprint database from remote
python main.py -d example.com --update-fingerprints

# Use custom fingerprint URL
python main.py -d example.com \
  --fingerprint-url https://your-server.com/fingerprints.yaml \
  --update-fingerprints
```

### Full Example (Bug Bounty)

```bash
python main.py \
  -d target.com \
  --subfinder \
  --threads 75 \
  --timeout 12 \
  --html \
  --json \
  --markdown \
  --min-confidence 40 \
  --discord-webhook https://discord.com/api/webhooks/... \
  --min-severity medium \
  -o ./results/target_com \
  --accept-legal \
  --verbose
```

## Adding Custom Fingerprints

Edit `fingerprints/fingerprints.yaml`:

```yaml
fingerprints:
  - service: "My Custom Provider"
    cname_patterns:
      - ".myprovider.io"
    body_fingerprints:
      - "This site doesn't exist"
      - "No such account"
    status_codes: [404]
    claimable: true
    difficulty: "easy"
    notes: "Register an account at myprovider.io with this subdomain"
    docs: "https://myprovider.io/docs/custom-domains"
    confidence_boost: 20
```

## Responsible Disclosure

When you find a vulnerability:

1. **Do not exploit it** — verify only that it exists
2. **Collect evidence** — use the generated `proof.json`
3. **Contact the organization** — via their security contact or bug bounty platform
4. **Give reasonable time** — typically 90 days before public disclosure
5. **Follow the program rules** — respect scope and reporting guidelines

Common disclosure channels:
- `security@domain.com`
- HackerOne / Bugcrowd / Intigriti program
- GitHub Security Advisory
- `/.well-known/security.txt`
